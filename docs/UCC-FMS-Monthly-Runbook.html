A. During the month — record & POST as transactions happen

Money received → Receipts / RV → record → Post
Not in the ledger until you Post it.
Payments made → Payments / PV → record → choose Post now on the save prompt
Imprest, assets, advances & project payments all go here, against the official UCC COA.
Clear all drafts → on Payments / PV, if the amber Draft badge shows, click ⚡ Post all drafts
A Draft PV is saved but NOT in any report.
Commitments (POs/contracts) → Commitments for budget control before spending.

B. Statutory & periodic (each month)

Payroll → Payroll → Run → Approve
Approve posts the salary journal (PAYE, SSNIT, net pay) to the ledger.
Depreciation → Asset Register → Run depreciation once for the month.
Remit withholdings → Withholding Payables → settle WHT / VAT / UCF to GRA when paid.
Fuel → Fuel & Vehicle → reconcile coupons issued vs stock; post procurement.

C. Month-end close — do in this order

1
Bank Reconciliation → Bank Reconciliation — match each statement line to the ledger.
2
Trial Balance → Reports → Trial Balance — confirm total Debit = total Credit.
3
Financial statements → generate SFP, Income & Expenditure, Cash Flow, Changes in Net Assets.
4
Budget Variance → Reports → Budget Variance — review actual vs budget; explain overspends.
5
Period Close → Period Close — lock the month & sign off so no further posting changes it.
6
Back up → Backups → Download Backup — keep an off-site copy.
THREE CONTROLS THAT MUST PASS BEFORE YOU CLOSE
No PVs left in Draft status
Trial Balance balances (Debit = Credit)
SFP presentation difference = 0 (Assets = Liabilities + Net Assets), and Cash Flow closing = SFP cash

D. Year-end (once per financial year, after the last month is closed)

  • Year-End Close → Year-End Close — posts the closing journals that carry the year's surplus/deficit into Accumulated Fund. Income & expense reset to zero for the new year; balance-sheet balances carry forward.
If a number looks wrong: never edit a posted entry by hand. Open it → Admin Edit → the system posts a reversal + a corrected entry (full audit trail). The reversal line in the ledger is normal — the net effect is only your correction.
